# 🎬 Cinema Demand Prediction — ML Microservice

A standalone **FastAPI + XGBoost** microservice that predicts movie show demand (score 0–1) used by the BookNView Node.js backend for dynamic discount pricing.

---

## 📂 Folder Structure

```
ml-service/
├── data/
│   └── cinema_demand.csv   # Training dataset (~500 rows)
├── model/
│   └── demand_model.pkl    # Trained model (generated by train_model.py)
├── predict_api.py          # FastAPI app
├── train_model.py          # One-time training script
├── requirements.txt        # Python dependencies
└── README.md
```

---

## 🚀 Setup & Run

### Prerequisites
- Python 3.9+
- pip

### 1 — Install dependencies

```bash
cd ml-service
pip install -r requirements.txt
```

### 2 — Train the model (one-time)

```bash
python train_model.py
```
This reads `data/cinema_demand.csv`, trains XGBoost, and saves `model/demand_model.pkl`.  
Expected output:
```
📂 Loading dataset ...
   Rows: 500 | Columns: [...]
🚀 Training XGBoost model ...
📊 Test MAE  : 0.0120
📊 Test R²   : 0.9940
✅ Model saved → .../model/demand_model.pkl
```

### 3 — Start the API server

```bash
uvicorn predict_api:app --reload --port 8000
```

API is now live at **http://localhost:8000**

## 📡 API Endpoints

### `POST /predict-demand`

Predicts the demand score for a cinema show.

**Request body**
```json
{
  "show_hour"          : 22,
  "day_of_week"        : 1,
  "seat_occupancy_pct" : 0.10,
  "movie_popularity"   : 0.25,
  "recent_bookings"    : 3
}
```

| Field               | Type  | Range   | Description                            |
|---------------------|-------|---------|----------------------------------------|
| `show_hour`         | int   | 0–23    | Hour of day the show starts            |
| `day_of_week`       | int   | 0–6     | 0 = Monday, 6 = Sunday                 |
| `seat_occupancy_pct`| float | 0.0–1.0 | Fraction of seats already booked       |
| `movie_popularity`  | float | 0.0–1.0 | Normalised popularity/rating score     |
| `recent_bookings`   | int   | ≥ 0     | Bookings made in the last 60 minutes   |

**Response**
```json
{
  "demand_score" : 0.17,
  "demand_level" : "LOW"
}
```

### `GET /health`
Returns `{ "status": "ok" }`.

### Interactive docs
Visit **http://localhost:8000/docs** for the Swagger UI.

## 🧪 Quick Test (curl)

```bash
# Low-demand scenario (expect score < 0.4)
curl -X POST http://localhost:8000/predict-demand \
  -H "Content-Type: application/json" \
  -d "{\"show_hour\":22,\"day_of_week\":1,\"seat_occupancy_pct\":0.10,\"movie_popularity\":0.25,\"recent_bookings\":3}"

# High-demand scenario (expect score > 0.6)

  -H "Content-Type: application/json" \
  -d "{\"show_hour\":19,\"day_of_week\":5,\"seat_occupancy_pct\":0.85,\"movie_popularity\":0.90,\"recent_bookings\":55}"
```

## 🔗 Integration with Node.js Backend

The Node.js backend calls this service from:
- `backend1/services/dynamicPricingService.js`
- Exposed via `POST /api/v1/pricing/ticket-price`

**Pricing rule applied in Node.js:**
- `demand_score < 0.4` **AND** showtime is within **60 minutes** → **20% discount**
- All other cases → base price (no price increase ever)
